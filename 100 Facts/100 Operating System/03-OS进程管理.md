## 中断
* 中断会导致处理器暂停当前正在执行的任务，并转向执行一个特定的处理程序（中断处理程序）。然后在处理完这些特殊情况后，处理器会返回到被打断的任务继续执行。
 * 中断是由计算机系统外部事件触发的，通常与硬件设备相关。中断的目的是为了及时响应重要事件而暂时中断 正常的程序执行。
* 典型的中断包括时钟中断、I/O设备中断（如键盘输入、鼠标事件）和硬件错误中断等。操作系统通常会为每种类型的中断分配一个中断处理程序，用于处理相应的事件。
## 进程、线程、协程定义
* 进程
	* 系统资源分配和调度的**基本单位**
	* 拥有独立的内存空间和系统资源
	* 每个进程通常有自己的地址空间、文件描述符、堆栈等
* 线程
	* 线程是程序执行的**最小**单位，是 CPU 调度的**基本单位**
	* 多个线程共享进程的内存和资源
	* 每个线程拥有独立的执行栈和寄存器
	* 线程切换进入内核态
* 协程
	* 用户态的轻量级线程，由程序控制调度，通常不依赖操作系统的调度器
	* 在用户态完成切换，不需要内核干预
## 设计线程协程的原因
* 资源
* 通信
* 切换开销
* 创建销毁
## 通信方式
### 进程间通信
* 管道（Pipe）：单向通信机制，一般用于父子进程之间的通信。
	* 比如ls | grep "txt" 前后分属两个进程，前面的输出作为后面的输入
- 消息队列（Message Queue）：允许不同进程间通过消息传递进行通信，消息队列是一个先进先出的数据结构，支持异步通信。
- 共享内存（Shared Memory） mmap， 将一块物理内存区域映射到虚拟地址空间，使得多个进程可以访问同一块内存。
- 信号量（Semaphore）
- 套接字（Socket）：可用于进程间在同一台机器或网络上的通信，支持基于 TCP/IP 协议的通信。
### 线程间通信
- 共享内存（Shared Memory）
- 条件变量（Condition Variable）
- 信号量（Semaphore）
- 互斥锁（Mutex）

## 进程状态
   - **创建状态（New）**：OS 为进程分配资源，初始化 PCB，还未加入调度队列;
   - **就绪状态（Ready）**：进程已分配完资源，等待被调度执行；
   - **运行状态（Running）**：进程正在被 CPU 执行；
   - **阻塞状态（Blocked）**：进程因等待某事件（如 I/O）暂时不能运行；
   - **终止状态（Terminated）**：OS 清理进程资源，撤销 PCB，进程执行完毕或异常退出。

## PCB Process Control Block
进程控制块，保存进程运行的现场环境
### 内容
#### 1. 进程识别信息 (Identity)
- **进程 ID (PID)：** 唯一的数字标识符，就像身份证号一样。
- **父进程 ID：** 用于构成进程树结构。
- **用户 ID (UID) 和组 ID (GID)：** 用于权限和安全控制。
#### 2. 处理器状态信息 (CPU State)
这是 PCB 中**最关键**的部分，也是实现 CPU 调度的基础。当一个进程（更准确地说是它的线程）被暂停运行时，OS 必须把它的运行状态保存下来，以便下次能从上次停下的地方继续。
- **程序计数器 (Program Counter)：** 指示下一条要执行的指令的地址。
- **CPU 寄存器值：** 各种通用寄存器、栈指针等在暂停时的内容。
- **进程状态字 (PSW)：** 包含条件码和控制位等。
#### 3. 进程调度信息 (Scheduling)
- **进程状态**
- **优先级：** 进程被选中的优先等级。
- **调度参数：** 例如，时间片大小、已占用的 CPU 时间等。
#### 4. 内存管理信息 (Memory)
这些信息用于实现进程的**地址空间**隔离。（通过[虚拟内存](02-OS内存管理.md)实现）
- **页表或段表指针** (现代OS)：这些指向了进程在虚拟内存和物理内存之间的映射关系数据结构。
#### 5. I/O 状态信息 (I/O & Resources)
- **已打开的文件列表：** 进程正在使用的所有文件句柄。
- **分配给该进程的 I/O 设备信息。**

## 孤儿进程&僵尸进程
### 孤儿进程
* 指一个进程的父进程已经退出或终止了，但是这个子进程本身还在运行
* 在 Linux 中，有一个 pid 为 1 的 init 进程，如果一个进程变成了孤儿进程，那么操作系统会把他的父进程重新指定为 init 进程。
### 僵尸进程
* 僵尸进程是指进程已经终止，但它的**退出状态**还没有被父进程回收（父进程通过调用wait()或waitpid()系统调用来获取这些信息，当附近成调用这些系统调用时，os 会把僵尸进程从进程表中移除）。操作系统保留了进程的退出信息（如退出码、进程 ID 等），但该进程的所有其他资源（如内存、CPU 时间等）已被释放。

## 进程调度算法（抽象出的算法逻辑）

### 批处理中的调度（古老且宏观的系统）
#### 先来先服务（FCFS）
- **描述**：按照作业到达的顺序来进行调度
- **优点**：易于实现，适用于长时间作业。
- **缺点**：**“长作业拖慢短作业”**。短作业可能会长时间等待，因为每个作业都必须等前一个作业完成才开始执行。
- **优化点**：适用于批处理任务较为简单的情况，但对响应时间要求高的任务不太适合。
#### 最短作业优先（SJF）
- **描述**：选择估计运行时间最短的作业先执行。作业的运行时间通常是系统根据历史经验或者任务的性质估计的。
- **优点**：可以最小化平均等待时间。
- **缺点**：**“长作业饿死”**，即如果短作业不断到来，长作业可能永远得不到调度。
- **优化点**：适用于短作业为主的环境，但不适用于不确定运行时间的情况。
#### 最短剩余时间优先（SRTF）
- **描述**：最短剩余时间优先（Shortest Remaining Time First）是最短作业优先的抢占式版本。它会选择剩余执行时间最短的进程。如果新的进程的剩余时间比当前运行进程的剩余时间更短，系统会抢占当前进程，调度新的进程。
- **优点**：可以最大程度地减少平均等待时间，尤其适合交互式应用。
- **缺点**：同样面临“饿死”问题，并且由于抢占特性，系统开销较大。
### 交互式系统中的调度（现代系统）
#### 时间片轮转调度（RR）
- **描述**：时间片轮转调度（Round Robin，RR）是一种抢占式调度算法。所有进程按 FCFS 排列，系统按时间片大小轮流调度进程，每个进程只能执行时间片长度的时间。时间片用完后，进程被挂起并放到队列末尾，调度下一个进程。
- **优点**：公平性高，每个进程都会按顺序轮流得到 CPU 时间。适用于多任务处理。
- **缺点**：如果时间片过长，实际上接近于 FCFS；如果时间片过短，频繁的上下文切换会增加系统开销。
#### 优先级调度
- **描述**：优先级调度算法根据每个进程的优先级来进行调度，优先级较高的进程会优先获得 CPU 时间。为了防止低优先级进程长时间得不到调度，系统通常会实施优先级提升策略，使等待时间长的进程逐渐获得更高的优先级。
- **优点**：适用于需要优先处理高优先级任务的场景。
- **缺点**：如果没有优先级提升机制，低优先级进程可能会长期得不到调度，导致**饥饿问题**。
#### 多级队列调度
- **描述**：多级队列调度是将系统中的进程分为多个优先级不同的队列。每个队列的调度策略可以不同。例如，前面队列使用较短的时间片轮转，后面队列使用较长的时间片轮转。进程会根据其性质和执行状态在队列之间移动。
- **优点**：结合了时间片轮转调度和优先级调度的特点，能够兼顾不同类型任务的需求。
- **缺点**：队列间的迁移策略可能会导致某些进程的执行被延迟。


## 调度算法的分类
### 分时调度
* 即时间片轮转，适用于交互式系统
### 实时调度
* 关注时效性和优先级，分为硬实时（必须在截止时间前完成）和软实时（尽量在截止时间前完成）
* 在实时操作系统（RTOS）中，RMS 和 EDF 较为常见
	* RMS（Rate Monotonic Scheduling 速率单调调度）
		* 事先定义好优先级
	* EDF（earliest Deadline First 最早解释时间优先）
		* 动态优先级，根据绝对截止时间来定义优先级


## 进程同步
### 定义
进程同步是指多个进程在执行过程中为了保证共享资源的一致性和正确性，而按照一定的规则协同执行的过程。**目标是**有序访问共享资源，避免竞态条件。
进程通信是手段，进程同步是目的。
### 同步机制
#### 信号量
   - 是一种整型计数器，用于控制多个进程对共享资源的访问。
   - 使用技术及表示资源数量，若资源数为 1，即为二元信号量。
   - 操作包括：
     - P 操作（等待）：将信号量减 1，若小于 0，则阻塞。
     - V 操作（释放）：将信号量加 1，唤醒等待进程。
#### 互斥锁
* 保证共享数据在同一时刻只能被一个线程访问，只能被同一个进程加解锁，适用于隔离资源。提供了所有权检查和优先级保护。
* 优先级保护即为，当高优先级和低优先级共享同个资源，当低优先级任务先占用该资源，那么该低优先级任务的优先级提高到和高优先级相同。
#### 条件变量
* 需要和互斥锁配合使用
* 在互斥锁的保护线，让线程等待的某个逻辑条件成立
#### 自旋锁
* 在等待时刻持续占用 CPU 轮询
####  事件
* 两种状态
	- **已激活（Signaled / Set）**：表示事件已发生，允许等待该事件的线程继续执行。
	- **未激活（Non-Signaled / Reset）**：表示事件尚未发生，尝试等待该事件的线程将被阻塞。
- 事件分类，根据触发后行为分类
	- 自动复位事件
		- 被激活时只唤醒一个等待的线程，并自动将事件状态重置为未激活
	- 手动复位事件
		- 被激活时所有等待该事件的现成都会被唤醒并执行，直到有线程显式调用 Reset 函数
