## 死锁
### 产生死锁的条件
* 互斥条件
	* 某个资源在同一时间只能被一个进程（或线程）占用，不能共享
* 请求并保持
	* 一个进程至少应该占有一个资源，并等待另一资源，而该资源被其他进程占有
* 不可剥夺
	* 资源不能被抢占，只有当持有资源的进程完成任务后，该资源才会被释放
* 循环等待
	* 存在一组进程形成了首尾相接的循环等待链
### 预防死锁
* 思路：破坏 4 个必要条件同时发生
	* 破坏请求并保持：要求进程一次性申请所有资源，不能满足就不分配
	* 破坏不可抢占：设置优先级，使得优先级高的线程可以抢占资源
	* 破坏循环等待：对资源编号，按序申请资源 - 最常用
### 避免死锁
* 在动态分配资源场景下做“安全性检测”，*确保不会进入*死锁状态
* 银行家算法用于死锁避免
### 检测死锁
* *允许*死锁发生，然后检测
* 死锁检测算法：
	* 资源分配图或等待图，如果有环路则有思索
* 检测时机：
	* 周期性检测
	* 资源紧张时
	* 长时间阻塞时
### 解除死锁
* 检测到死锁后，干预打破死锁
* 方法
	* 终止进程
	* 资源剥夺
		* 强制从某些进程中剥夺资源，分配给其他进程，打破循环等待
	* 回滚
		* 将某些进程回滚到之前的安全状态；需要系统支持检查点机制

## 资源管理
### 文件描述符
* file descriptor
* 操作系统为每一个打开的文件或者 IO 资源分配的一个非负整数标识
#### FD的分类
* FD 是内核用来标识和管理各种 IO 资源的统一接口
	- 普通文件（例如 `/home/a.txt`）
	- 目录
	- [管道（pipe)](03-OS 进程管理)
	- 套接字（socket）
	- 终端（stdin/stdout）
	- 设备文件（如 `/dev/null`、串口、网卡）
#### 本质
* 每个进程在内核都有一张fd表，每一项fd都指向一个”打开文件对象“（open file object）
```html
进程（用户态）
┌──────────────┐
│ 文件描述符表                       │
├───────┬──────┤
│ fd=0  │ -> stdin（标准输入）       │
│ fd=1  │ -> stdout（标准输出）     │
│ fd=2  │ -> stderr（标准错误）     │
│ fd=3  │ -> socket (127.0.0.1:8080)│
│ fd=4  │ -> /home/a.txt           │
└───────┴──────┘
```

- 执行 `open()`、`socket()`、`pipe()` 等系统调用时，内核：
    1. 分配一个空闲的 fd 值（最小的没被占用的整数）；
    2. 在进程的 fd 表中建立一条记录；
    3. 把这个 fd 返回给用户程序。
- 之后调用 `read(fd, ...)`、`write(fd, ...)`、`close(fd)` 时，  内核会根据这个 fd 找到对应的对象和驱动程序，完成 I/O 操作。

#### 生命周期
1. 分配
	1. 调用 oepn 等
2. 使用
	1. 使用 read 等
3. 释放
	1. close(fd)
