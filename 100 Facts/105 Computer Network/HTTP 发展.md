## HTTP
* 定义
	* 无状态、明文传输的协议
* 特点
	* 无状态：每个请求独立处理，不依赖于之前的请求。
    - 无加密：数据以明文形式传输，容易受到中间人攻击。
    - 基于请求/响应模型：客户端发起请求，服务器响应。

## HTTP/1.0
* 概述
	* 引入了请求/响应模型，每个请求和响应都需要单独建立和断开连接
* 特点
	* **短连接**：每个请求都需要建立一个独立的 TCP 连接，连接建立和断开需要耗费时间，导致高延迟。
	- **无持久连接**：每个请求需要单独的连接，不支持复用。

## HTTP1.1
* 概述
	* 在 1.0 基础上进行优化，引入持久连接和其他性能提升
* 特点
	* **长连接**：多个请求可以复用同一个 TCP 连接，减少了每次建立连接的开销。
	- **管道化**：允许多个请求同时发送，但请求之间仍然是顺序处理的。
	- **改进的缓存机制**：支持更多的缓存控制选项，如 `Cache-Control`

## 队头阻塞
* 定义
	* 队头阻塞发生在多路复用连接中，当一个队列中的第一个请求或数据包被阻塞时，所有后续的请求或数据包都必须等待前一个请求或数据包的完成才能继续处理。即使这些后续的数据包本身没有问题，也需要等待先到的阻塞数据包处理完成。

## HTTP/2.0
* 特点
	* **二进制协议**：相较于 HTTP/1.x 的文本协议，HTTP/2 使用二进制帧进行数据传输，减少了协议的复杂度和带宽消耗。
	- **多路复用**：允许在一个连接上并行处理多个请求和响应，具体表现为一个 stream 分为不同的 frame
	- **头部压缩**：通过 HPACK 算法压缩 HTTP 头部，减少了冗余数据。
	- **服务器推送**：服务器可以主动推送资源到客户端，减少客户端的请求次数。


## HTTP/3.0

- **概述**：基于 QUIC 协议，旨在减少连接建立时间和提高传输效率。
- **特点**：
    - **基于 QUIC 协议**：基于 UDP 的 QUIC 协议，减少了传统 TCP 的连接建立延迟。
    - **快速连接建立**：QUIC 支持 0-RTT 和 1-RTT 的连接建立方式
    - **多流传输**：QUIC 允许多个独立的数据流并行传输，避免了 HTTP/2 中的队头阻塞问题。
    - **连接迁移**：QUIC 支持在网络环境变化时保持连接，例如从 Wi-Fi 切换到移动数据。
        

## QUIC 协议

- **概述**：QUIC（Quick UDP Internet Connections）是由 Google 提出的协议，旨在改善网络传输性能，基于 UDP 实现。
- **特点**：
    - **基于 UDP**：QUIC 利用 UDP 提供更低的延迟，并在应用层实现拥塞控制、流量控制和重传机制。
    - **集成 TLS 加密**：QUIC 集成了 TLS 加密，减少了传统 HTTPS 中两次握手的延迟。
    - **独立的数据流**：QUIC 允许多个独立的数据流，并且数据流之间互不干扰，解决了队头阻塞问题。
    - **连接迁移**：QUIC 使用连接 ID 标识连接，使得在网络环境发生变化时（如切换网络）仍能保持连接不中断。
## HTTPS (安全超文本传输协议)

- **概述**：HTTPS 是在 HTTP 基础上加入 SSL/TLS 协议进行加密后的版本，确保数据的机密性、完整性和身份验证。
- **特点**：
    - **加密传输**：通过 SSL/TLS 加密数据，确保数据不会被窃取或篡改。
    - **身份验证**：使用服务器的数字证书验证其身份，防止中间人攻击。
    - **数据完整性**：数据传输过程中使用校验机制，确保数据没有被篡改。
### HTTPS 加密过程
1. **客户端发起请求**：客户端向服务器发起 HTTPS 请求，包含支持的 SSL/TLS 版本、加密算法列表和客户端生成的随机数。
2. **服务器响应**：服务器返回选择的 SSL/TLS 版本、加密算法、服务器的数字证书和服务器生成的随机数。
3. **验证证书**：客户端验证服务器的证书是否有效，是否由受信任的 CA 签发，且域名是否匹配。
4. **密钥交换**：客户端生成一个会话密钥，并使用服务器的公钥进行加密后发送给服务器，服务器使用私钥解密，获取密钥。
5. **生成会话密钥**：客户端和服务器使用客户端和服务器生成的随机数、会话密钥，通过对称加密算法生成会话密钥。
6. **加密通信**：客户端和服务器使用会话密钥对通信内容进行加密和解密，开始安全的数据传输。

* 非对称加密算法用于 安全的交换密钥，例如 RSA 算法
* 对称加密算法用于 在数据传输时加解密，例如 AES 算法

* 非对称加密主要用于安全交换密钥，对称加密用于加密大量数据，是因为他加解密速度快。
