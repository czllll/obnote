 1. `final` 修饰变量
	- 局部变量：该变量在初始化之后不能再被赋值。
	- 成员变量：通常表示常量，即该变量一旦赋值，就无法再修改。
	- 常量：通常 `final` 与 `static` 一起使用，来定义常量，常量在*类加载*时初始化，并且一旦初始化后无法修改。
 2. `final` 修饰方法
	* 表示该方法 **不能被子类重写**。
	* 这保证了方法的实现不可变，通常用于那些不希望被修改的方法，确保行为的一致性。

 3. **`final` 修饰类**
	* 当 `final` 修饰类时，表示该类 **不能被继承**。
	* 这意味着没有任何类能够继承自一个 `final` 类，这常用于设计那些不能被修改的类，例如 `String` 类和一些工具类。
 4. `final` 在构造函数中的使用
	* 构造函数不能被 `final` 修饰，因为构造函数不能被继承。然而，在构造函数中使用 `final` 修饰变量是非常常见的，尤其是在确保类的不可变性时。
