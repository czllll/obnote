* 所谓限流器，即在指定时间内，限制能处理的请求为指定数量。这就要求我们初始化限流器的两个参数，一个是*给定时间*，一个是*给定最大处理数量*。
* 我们这里实现一个 RateLimiter 类进行限流模拟，首先定义一个 RateLimiter 接口，主要有三个方法
	* 尝试获取许可
	* 尝试获取指定数量许可
	* 重置限流器
```java
public interface RateLimter{
	boolean tryAcquire();
	boolean tryAcquire(int premits);
	void reset();
}
```
## 并发问题
限流器作为限制流量的中间件，肯定是会被并发调用的，因此我们在设计时就应该考虑并发请求的情况。常见的并发改造有如下几种方式
1. 加锁 [[深入理解synchronized]]：直接给方法加锁
2. CAS
3. [[深入理解ReentrantLock]]：用显式锁包裹最小临界区
## 计数器方法
### 固定窗口
* 在一个固定的时间窗口（如 1s）内维护请求数，超过阈值就拒绝
* **缺点**
	* 边界效应严重：某个短时区间的实际流量可能超过设定阈值；即 0.9-1.1s 不到 1s 的时间可能有 200 的请求
* **实现**
```java
package rateLimiter;  
  
import org.springframework.aop.scope.ScopedProxyUtils;  
import rateLimiter.RateLimiter;  
  
public class FixedWindowRateLimiter implements RateLimiter {  
    // 窗口最大请求  
    private final long maxReq;  
    // 窗口大小  
    private final long windowSize;  
    // 当前窗口开始时间  
    private long windowStartTime;  
    // 当前窗口已用请求数  
    private long reqCount;  
  
    public FixedWindowRateLimiter(long maxReq, long windowSize) {  
        if(maxReq < 0 || windowSize < 0) throw new IllegalArgumentException("参数不合法");  
        this.maxReq = maxReq;  
        this.windowSize = windowSize;  
        this.windowStartTime = alignTime(System.currentTimeMillis());  
        this.reqCount = 0;  
    }  
  
    private long alignTime(long time){  
        return time / windowSize * windowSize;  
    }  
  
    @Override  
    public boolean tryAcquire() {  
        return tryAcquire(1);  
    }  
  
    @Override  
    public boolean tryAcquire(int permits) {  
        // 判断参数合法性  
        if(permits <= 0) throw new IllegalArgumentException("参数不合法");  
        if(permits > maxReq) return false;  
  
        // 拿当前窗口  
        long curWindowStartTime = alignTime(System.currentTimeMillis());  
        if(curWindowStartTime != windowStartTime) {  
            // 新建窗口  
            windowStartTime = curWindowStartTime;  
            reqCount = 0;  
        }  
        if(reqCount + permits <= maxReq) {  
            reqCount += permits;  
            return true;  
        }  
  
        return false;  
    }  
  
    @Override  
    public void reset() {  
        windowStartTime = alignTime(System.currentTimeMillis());  
        reqCount = 0;  
  
    }  
  
    public long getRemainReqCount(){  
        if(alignTime(System.currentTimeMillis()) != windowStartTime) return maxReq;  
        return maxReq - reqCount;  
    }  
  
}  
  
class Main{  
    private static void testFixedWindow() throws InterruptedException {  
        System.out.println("3. 固定窗口算法演示:");  
        System.out.println("配置: 窗口大小=2秒, 最大请求数=3");  
  
        FixedWindowRateLimiter fixedWindow = new FixedWindowRateLimiter(3, 2000);  
  
        // 在窗口内发送请求  
        for (int i = 1; i <= 5; i++) {  
            boolean allowed = fixedWindow.tryAcquire();  
            System.out.println("发送五个请求，请求"+i+" "+allowed);  
            System.out.println("还能发个" + fixedWindow.getRemainReqCount() + "请求");  
        }  
  
        System.out.println("等待窗口重置...");  
        Thread.sleep(2100);  
  
        System.out.println("还能发个" + fixedWindow.getRemainReqCount() + "请求");  
  
        System.out.println();  
    }  
  
    public static void main(String[] args) throws InterruptedException {  
        testFixedWindow();  
    }  
}
```

### 滑动窗口
* 固定窗口和滑动窗口都是的具体实现
	* 计数器是用计数器来统计一段时间窗口的请求数量，这个时间窗口可以是固定窗口也可以是滑动窗口

## 令牌桶
* **思想**
	* 系统以固定速率往桶中放令牌，每次请求需要 1 个令牌，没有令牌则拒绝或排队；同有容量上限，令牌可累计形成突发
## 漏桶
* **思想**
	* 系统像一个底部漏水的桶，水流入桶代表外部请求到达，桶按固定速率漏水；桶满时新水溢出（请求被丢弃或拒绝）
