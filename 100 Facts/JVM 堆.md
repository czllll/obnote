## 对象
```ascii
+-------------------+
|     Mark Word     |   ← 哈希码 / GC 年龄 / 锁信息 / 偏向线程ID
+-------------------+
|   Klass Pointer   |   ← 指向类元数据 (Klass)，确定对象的类型
+-------------------+
|   (Array Length)  |   ← 仅数组对象有，用于记录数组长度
+-------------------+
|  Instance Data    |   ← 普通对象的实例字段 / 数组对象的元素
+-------------------+
|     Padding       |   ← 对齐填充 (保证对象大小是 8 或 16 的倍数)
+-------------------+
```
## 对象头
JVM 在给对象分配内存时，会在对象的起始部分留一块空间做对象头（Header）
### 结构
- **Mark Word**：哈希码、GC 年龄、锁标志位、指针（锁相关）
- **Klass Pointer**：指向类元数据（方法区 / Metaspace）
- **数组长度**（如果是数组对象）

#### Mark word
mark word 是对象头的一个可复用字段，存储对象在运行时的各种信息。
##### 结构
mark word 的内容不是固定的，而是根据对象状态（无锁、偏向、轻量级锁、重量级锁、GC）来解释
**字段以及作用**
* hashCode
	* 对象的 identity hashCode（懒加载生成）
* GC Age
	* GC 分代年龄（对象在 Survivor 区复制的次数）
* 锁标志位
	* 当前锁的状态（无锁/偏向/轻量级/重量级/GC）
		**01**：无锁 / 偏向锁，是否偏向由 bias bit 决定：bias=0 → 无锁bias=1 → 可偏向/偏向锁
		**00**：轻量级锁，对象头存放指针，指向线程栈帧中的 Lock Record
		**10**：重量级锁，对象头存放指针，指向本地内存中的 ObjectMonitor
		**11**：GC 标记（Marked）垃圾收集时使用
* 偏向线程ID
	* 偏向锁模式下，记录持有锁的线程 ID
* Epoch
	* 偏向锁时间戳，用于批量撤销偏向
* 指针
	* 指向 Lock Record（轻量级锁）或 ObjectMonitor（重量级锁）
* 偏向位(bias bit)
	* 1 表示可偏向，0 表示不可偏向
* GC 标记
	* GC 过程中的特殊标记
