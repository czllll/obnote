JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。
# 特性
* **运行时类信息访问**
* **动态对象创建**
	* 使用反射API动态创建对象实例，即使在编译时不知道具体的类名
	* `Class`类的`newInstance()`方法 或 `Constructor`对象的`newInstance()`方法实现的
* **动态方法调用**
	* 在运行时动态地调用对象的方法，包括私有方法
	* 通过`Method`类的`invoke()`方法实现，允许你传入对象实例和参数值来执行方法
* **访问和修改字段值**
	* 即使是私有的
	* 通过`Field`类的get()和set()方法完成的。

# Class 类
* **内容**：保存类的所有信息（成员变量、方法、构造器等）
* **范围**：每个类（以及数组类型和基本数据类型，包括void）在JVM中都有且仅有一个Class对象。
* **位置** #存疑
	* 普通类的Class对象来自于编译后的.class字节码文件，在JVM中通过类加载过程创建Class对象
	* 数组等基本数据类型内置在JVM中，JVM自动创建Class对象，不依赖[[类加载]]
* **构造函数**：是私有的，Class对象只能由JVM创建和加载

# 使用
## 获取Class对象
在类加载时， JVM会创建一个Class对象
**方式**：
- 根据类名：类名.class
- 根据对象：对象.getClass()
- 根据全限定类名：Class.forName(全限定类名)
