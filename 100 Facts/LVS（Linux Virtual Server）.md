LVS是一个运输层L4负载均衡解决方案，处理IP地址和端口号，运行在LInux系统的内核中。

## 基本架构
- **LVS 调度器**：负责接收客户端请求并将其转发到后端的服务器。调度器通常部署在集群的前端，它通常是一个运行在 Linux 内核中的 负载均衡模块。
- **后端服务器**：实际处理请求的服务器，通常是一组 Web 服务器、数据库服务器或其他类型的应用服务器。

## 工作原理
LVS 基于负载均衡算法，通过 目标 IP 地址（VIP） 和 端口号 来决定如何将流量转发给后端服务器。
- **虚拟 IP 地址（VIP）**：客户端向这个 IP 地址发起请求，LVS 负载均衡器接收这个请求并决定将请求转发到哪一台后端服务器。
- **真实 IP 地址（RIP）**：LVS 将请求转发到后端服务器的实际 IP 地址。

## LVS 支持的工作模式

### NAT 模式（Network Address Translation）
- 在 NAT 模式下，LVS 调度器会使用 [[DNAT（目的地址转写）]]修改数据包的目标地址，将 虚拟 IP 地址（VIP） 转换为后端服务器的 真实 IP 地址（RIP），然后将请求转发给后端服务器。
- 后端服务器处理请求后，响应会通过 LVS 再返回给客户端。这个过程中，LVS 扮演了 中间代理 的角色。
- 这种模式通常用于 后端服务器使用*私有 IP* 地址 的情况。

### DR 模式（Direct Routing）
- 在 DR 模式下，LVS 不修改数据包的源地址和目标地址，而是修改数据包的目标 MAC 地址，直接转发到后端服务器，而且后端服务器的响应直接返回给客户端，不经过 LVS 转发。
- 这种模式适用于 后端服务器和 LVS 位于*同一子网* 的情况，因为它允许数据包直接到达目标服务器，从而减少延迟，提高性能。

### TUN 模式（IP Tunneling）
- TUN 模式通过 [[IP 隧道]]将流量从 LVS 转发到后端服务器。LVS 会将数据包通过一个隧道转发到后端服务器，后端服务器直接向客户端返回响应，适用于 *跨子网* 的情况。
- 在这个模式下，LVS 和后端服务器之间通过隧道进行通信，适合跨网络的负载均衡。
