* file descriptor
* 操作系统为每一个一打开的文件或者 IO 资源分配的一个非负整数标识
## FD的分类
* FD 是内核用来标识和管理各种 IO 资源的统一接口
	- 普通文件（例如 `/home/a.txt`）
	- 目录
	- [管道（pipe)](进程间通信)
	- 套接字（socket）
	- 终端（stdin/stdout）
	- 设备文件（如 `/dev/null`、串口、网卡）
## 本质
* 每个进程在内核都有一张fd表，每一项fd都指向一个”打开文件对象“（open file object）
```html
进程（用户态）
┌──────────────┐
│ 文件描述符表                       │
├───────┬──────┤
│ fd=0  │ -> stdin（标准输入）       │
│ fd=1  │ -> stdout（标准输出）     │
│ fd=2  │ -> stderr（标准错误）     │
│ fd=3  │ -> socket (127.0.0.1:8080)│
│ fd=4  │ -> /home/a.txt           │
└───────┴──────┘
```

- 执行 `open()`、`socket()`、`pipe()` 等系统调用时，内核：
    1. 分配一个空闲的 fd 值（最小的没被占用的整数）；
    2. 在进程的 fd 表中建立一条记录；
    3. 把这个 fd 返回给用户程序。
- 之后调用 `read(fd, ...)`、`write(fd, ...)`、`close(fd)` 时，  内核会根据这个 fd 找到对应的对象和驱动程序，完成 I/O 操作。

## 生命周期
1. 分配
	1. 调用 oepn 等
2. 使用
	1. 使用 read 等
3. 释放
	1. close(fd)
