* JVM 内存结构是硬件和操作系统层面的抽象；
* JMM 定义了 Java 内存操作的规则，并依赖于 JVM 内存结构来实现。

### JVM 内存结构

**JVM 内存结构**（JVM Memory Structure）指的是 Java 虚拟机在运行程序时，将数据划分成不同的区域进行存储和管理。它是一个**物理概念**，属于运行时数据区，是 JVM 运行时对内存的划分。

JVM 内存结构主要包括以下几个部分：

- **程序计数器（Program Counter Register）：** 存储当前线程执行的字节码指令地址。
    
- **Java 虚拟机栈（Java Virtual Machine Stacks）：** 存储方法执行时的局部变量、操作数栈、动态链接、方法出口等信息。每个方法执行都会创建一个栈帧。
    
- **本地方法栈（Native Method Stack）：** 存储本地方法（Native Method）的信息。
    
- **Java 堆（Java Heap）：** 存储所有对象实例和数组。所有线程共享。
    
- **方法区（Method Area）：** 存储已被加载的类信息、常量、静态变量、即时编译器编译后的代码等。所有线程共享。
    
### JMM 内存结构

**JMM 内存结构**（Java Memory Model）指的是 Java 内存模型，它是一个**抽象概念**，是 Java 语言规范中定义的一套规则，用于屏蔽各种硬件和操作系统的内存访问差异，确保 Java 程序在不同平台上都能保持一致的内存可见性。

JMM 主要关注的是在多线程环境下，变量如何被读写、一个线程对共享变量的修改如何被另一个线程看到。它定义了以下两个核心概念：

- **主内存（Main Memory）：** 所有线程共享的公共内存，可以看作是 JVM 堆内存中存储的对象数据。所有变量都存储在这里。
    
- **工作内存（Working Memory）：** 每个线程独有的私有内存，存储了该线程操作的共享变量的副本。它对应着 CPU 的高速缓存（如 L1, L2 Cache）。

### 两者关系

JVM 内存结构和 JMM 内存结构之间的关系可以用下图来简单描述：

```
       JVM 内存结构 (运行时数据区)
       +-------------------------+
       |                         |
       |  堆 (Heap)              | <-----> 主内存 (Main Memory)
       |  (共享，存储对象)       |          (所有共享变量)
       |                         |
       +-------------------------+
       |                         |
       |  虚拟机栈 (VM Stacks)   | <-----> 工作内存 (Working Memory)
       |  (线程独享，存储局部变量等) |          (线程私有，变量副本)
       |                         |
       +-------------------------+
```

**简单来说，它们的关系是：**

- **物理 vs. 抽象：** **JVM 内存结构** 是**物理**上的内存划分，是实际存储数据的区域。**JMM** 是**抽象**的规则，它定义了多线程如何通过主内存和工作内存进行数据交互。
    
- **依赖关系：** JMM 的主内存和工作内存概念是建立在 JVM 内存结构上的。JMM 中的**主内存**可以对应到 JVM 的**堆**，而 JMM 中的**工作内存**可以对应到 JVM **虚拟机栈**中线程私有的部分，以及 CPU 的高速缓存。
    
- **目的不同：** JVM 内存结构主要关注内存的**存储和管理**，解决的是“数据放在哪里”的问题。JMM 主要关注多线程环境下的**可见性、有序性和原子性**，解决的是“数据如何同步和交互”的问题。
