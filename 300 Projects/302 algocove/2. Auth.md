# 需求
## 功能性需求
1. 注册
	1. 邮箱验证码+密码注册
2. 登录
	1. 使用邮箱+密码
3. 支持第三方如google / github登录

# 方案
* 接口
	* 发送验证码接口
		* `/api/auth/verification/send`
	* 注册或登录接口
		* `/api/auth/registerOrLogin`

* [[OAuth]] 方案
	* 第三方作用
		* 提供用户信息
			* username
			* githubId
		* 认证通过后，调用系统后端（satoken）生成token给前端
	* 流程
		* 用户点击github登录
			* 跳转认证界面，后端跳转with state
		* 用户点击授权
			* github 回调callback?code=xxx & state
			* 后端回调接口
				* 验证state， 防止crsf
				* 然后我的后端再进行请求？
					* 拿 code 换 access_token
				* 然后后端再进行请求
					* access_token 拉 GitHub 用户信息
					* 有了用户信息， 注册用户，使用satoken维护登录？
