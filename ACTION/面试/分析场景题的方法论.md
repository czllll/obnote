# 场景题分析方法论

## 第一步：需求还原

1. 明确业务目标
2. 梳理核心功能
3. 识别关键约束条件
4. 还原系统交互流程

## 第二步：整体架构设计

### 功能性需求拆解

- 列出系统核心功能
- 分析每个功能的输入输出
- 识别功能间的依赖关系

### 非功能性需求分析

- 性能要求
- 可扩展性
- 可用性
- 一致性
- 安全性

## 第三步：技术方案设计

### 架构选型

1. 选择合适的架构模式
    
    - 单体架构
    - 微服务架构
    - 分布式架构
2. 服务拆分
    
    - 按业务边界划分
    - 考虑服务职责单一性
    - 评估服务间调用成本

### 数据模型设计

1. 存储方案选择
    
    - 关系型数据库
    - NoSQL数据库
    - 缓存数据库
2. 关键表结构设计
    
    - 主键设计
    - 索引策略
    - 冗余与否

## 第四步：关键技术点

### 高并发处理

1. 缓存策略
2. 异步处理
3. 消息队列
4. 分布式锁

### 数据一致性

1. 事务管理
2. 幂等设计
3. 补偿机制

### 性能优化

1. 读写分离
2. 分库分表
3. 索引优化
4. 缓存命中率

## 第五步：细节实现

1. 关键流程伪代码
2. 异常场景处理
3. 边界条件验证

## 第六步：系统扩展性

1. 水平扩展能力
2. 技术演进路径
3. 系统解耦
4. 监控与告警

## 总结与反思

- 是否满足原始需求
- 是否具备可扩展性
- 是否存在明显的性能瓶颈
- 是否有可优化空间
