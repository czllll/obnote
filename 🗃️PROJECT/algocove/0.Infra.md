# 接口日志采集stater
* 目的
	* 自定义注解，支持在接口上添加注解，然后就有日志了。
* 方案
	* 自定义注解写在一个stater里

* 定义注解的方法
	* 写一个注解类
		* 属性定义语法
* 定义一个切面类
	* 定义拦截点（切点）
		* @Pointcut
	* 编写通知逻辑

|注解|说明|
|---|---|
|`@Before`|方法执行前调用|
|`@After`|方法执行后调用（无论成功失败）|
|`@AfterReturning`|方法返回后调用（成功）|
|`@AfterThrowing`|方法抛异常后调用|
|`@Around`|包裹方法执行（可以控制是否执行）|
* 要拿到执行方法的一些数据，所以使用Around
* 在切面类里使用jointPoint拿到一些数据
	* JoinPoint 代表“某一个被 AOP 拦截的方法调用现场”

|方法|说明|
|---|---|
|`getTarget()`|被代理的目标对象（比如 Controller 实例）|
|`getArgs()`|方法参数（Object[]）|
|`getSignature()`|方法签名（可以拿到方法名）|
|`getKind()`|连接点类型（通常是 method-execution）|
|`getThis()`|代理对象（通常是 CGLIB 动态代理类）|
一个starter流程
1. 写逻辑
	1. 切面和注解
2. 写配置
	1. 写autofig
	2. 添加自动配置声明（告诉 Spring Boot 你的配置类在哪里）

# Side 知识
* 怎么写自定义注解？
	* 定义一个注解类（三个注解）
		* @Target(ElementType.METHOD) 注解作用范围
		* @Retention(RetentionPolicy.RUNTIME)  // 保留策略，指定注解保留到什么时候（源码、class 文件、运行时）
		* @Documented  // 生成 javadoc 时包含
	* 编写切面（AOP）拦截注解逻辑
		* 上面有 切点 + 通知
