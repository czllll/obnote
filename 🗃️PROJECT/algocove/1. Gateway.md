# 背景
* 采用spring cloud gateway进行路由
# 职责
1. 根据URI请求转发到*微服务*，注册中心做服务发现
2. 拦截请求， 验证Token是否存在， 是否过期
3. 请求改写 / Header 注入：将 Token（通常是 JWT）中的 `userId`、`role` 等透传给下游服务（比如加到 Header）。
4. 请求预处理
	1. 请求限流（风控，放刷等）
	2. IP过滤
	3. 日志记录（请求路径，耗时等）
5. 全局异常处理
6. 服务降级，熔断
	1. 当下游服务不可用时返回默认值或友好提示
	2. 可配合 Sentinel / Resilience4j 实现
7. 统一设置CORS
# 方案
1. 在yml文件中规定URI-> 微服务的lb路径
2. 集成sa-token做checkLogin(), 放行登录注册，验证码等接口
	1. 实现一个sa-token类的一个实例SaReactorFilter，注册到容器
3. userId透传
	1. 实现全局过滤器，使用stpUtil拿到loginid 然后透传到下游服务


# side 知识
* POM 文件中的BOM 是指 bill of material 物料清单，也就是依赖管理，是一种特殊的 `POM` 文件，用于**集中管理一组相关依赖的版本号**。
```
<type>pom</type> 表示这个依赖是一个配置中心而非jar
<scope>import</scope> 只导入BOM中的版本管理规则
```
